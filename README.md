# Тестовые задания, SOP Analytics

Здесь три тестовых задания под стажёрские позиции в нашей команде. Мы делаем аналитическую платформу на sopanalytics.miem.tv: студенты оставляют отзывы о преподавателях ВШЭ, администрация смотрит инсайты. В базе больше 178 тысяч отзывов и 7 тысяч преподавателей.

Все три задания независимы друг от друга. Их делают разные люди на разные позиции. Выбери направление и открой нужный файл.

## Направления

**[Backend](./test-backend-go.md)**

Go плюс PostgreSQL. Мини-сервис с регистрацией, JWT-авторизацией, миграциями, CRUD, rate limiting, Docker Compose. 10-12 часов. Если ты питонист, можно сделать на FastAPI или Flask, детали внутри.

**[Frontend](./test-frontend.md)**

React 18 или 19, Vite, plain CSS. Каталог товаров с авторизацией на базе публичного API `dummyjson.com`. Логин с JWT и рефрешем, поиск, фильтры, своя SVG-диаграмма, корзина, тёмная тема. 8 часов. Свой бэкенд не нужен.

**[DevOps](./test-devops.md)**

Ubuntu-сервер, Docker Compose с тремя сервисами (app, postgres, nginx), SSL, firewall, бэкапы. 4-6 часов. Приложение даём готовое (40 строк Go) или берёшь любое своё.

## Как мы смотрим тестовые

Сначала прогоняем по инструкции из README. Если не стартует по вашим же шагам на чистой машине за 10-15 минут, это плохой сигнал. Если стартует, идём в код.

Дальше смотрим на безопасность. Хардкод паролей, plain SQL, отсутствие валидации, пропущенные проверки токенов, ключи прямо в репозитории. Всё это минусы. Мы правда смотрим.

Смотрим на структуру. Один файл на 3000 строк и двадцать микросервисов ради CRUD это обе крайности. Хочется понять, как ты делишь код по смыслу.

Смотрим внимание к деталям. Healthcheck настоящий или заглушка. Ошибка 500 крашит UI или показывает осмысленное сообщение. Коммиты осмысленные или все «update». Из таких мелочей складывается впечатление.

Смотрим документацию. README реально работает или написан «для галочки». Если на интервью попросим рассказать своё решение, сможешь по памяти, или будешь читать свой же текст.

## Про AI

LLM использовать можно, мы сами пользуемся. Но пиши свой код. На интервью мы разбираем твоё тестовое, и «это мне написал ChatGPT» не ответ на вопрос «почему здесь такое решение». Честное «вот тут подсмотрел, сам бы не догадался» всегда лучше.
